### О проекте

**Remote device message analyzer** - десктопное приложение на C++/Qt для анализа и расшифровки протоколов обмена удалённых устройств.

### Ключевые возможности

- Автоматическое распознавание
- Декодирование параметров в понятном человеку виде
- Валидация данных согласно спецификаиям протоколов
- Интуитивно понятный графический интерфейс
- Подробное логирование ошибок и результатов анализа

### Сборка из исходников

#### Зависимости

- cmake
- qt5

#### Сборка
```bash

git clone https://github.com/alexandrDuckyBobrov/remote-device-message-analyzer.git
cd remote-device-message-analyzer

cd build

cmake .. -DCMAKE_BUILD_TYPE=Release

make clean

make -j$(nproc)

```

#### Запуск

Из папки /build

```bash
./untitled
```

### Формат ввода данных

Сообщения вводятся в шестнадцатеричном формате, байты разделяются пробелами

Примеры корректных сообщений:

**Коршун**
- `3A 46 30 31 31 30 53 32 43 30 38 21`

**Дрофа**
- `FF FF 13 88 0A 64 EF EF`

### Спецификация протоколов

#### Протокол "Коршун"

Вид сообщения:

`<НС><Данные><КС>`

| Поле        | Длина  | Значение                    |
| ----------- | ------ | --------------------------- |
| НС (Начало) | 1 байт | `0x3A` (`:`)                |
| Данные      | N байт | Параметры: F(4), S(1), C(2) |
| КТ (Конец)  | 1 байт | `0x21` (`!`)                |

Коды параметров:

- `F` (0x46) - частота: 100-5000, шаг 10
- `S` (0x53) - чувствительность: 1=мин, 2=сред, 3=макс
- `C` (0x43) - канал: 1-16

#### Протокол "Дрофа"

Вид сообщения:

`<НС><F><M><P><КТ>`

| Поле        | Длина   | Значение                  |
| ----------- | ------- | ------------------------- |
| НС (Начало) | 2 байта | `0xFFFF`                  |
| F           | 2 байта | частота (5000-50000 Гц)   |
| M           | 1 байт  | режим (0x01/0x02/0x0A)    |
| P           | 1 байт  | мощность (0-100%, шаг 10) |
| КТ (Конец)  | 2 байта | `0xEFEF`                  |
